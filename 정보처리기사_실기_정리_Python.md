## 문제 12 — 함수 안의 대입과 반환값

```python
def calc(x, y):
    x *= 3
    y /= 3
    print(x, y)
    return x

a, b = 3, 12
a = calc(a, b)
print(a, b)
```

### 정답

```text
9 4.0
9 12
```

### 풀이 과정

호출 직전에는 `a = 3`, `b = 12`다. `calc(a, b)`를 호출하면 `x`와 `y`는 각각 `3`, `12`라는 값을 받아 함수 안의 지역 이름이 된다.

|순서|함수 안 값|이유|
|---|---|---|
|호출 직후|`x = 3`, `y = 12`|`a`, `b`의 값이 인수로 전달된다.|
|`x *= 3` 뒤|`x = 9`|`x = x * 3`과 같다.|
|`y /= 3` 뒤|`y = 4.0`|Python 3의 `/`는 실수 나눗셈이므로 `4.0`이다.|
|첫 `print`|`9 4.0`|함수의 지역값을 출력한다.|
|반환 뒤|`a = 9`, `b = 12`|반환값 `x`만 `a`에 대입되고, 바깥의 `b`는 바뀌지 않는다.|

정수 `int`는 불변 객체이므로 `x *= 3`, `y /= 3`은 바깥 변수 자체를 수정하는 동작이 아니다. 마지막 출력은 따라서 `9 12`다.

## 문제 18 — `range`와 반복 변수의 범위

```python
hap = 0
for i in range(1, 11):
    hap += i
print(i, hap)
```

### 정답

```text
10 55
```

### 풀이 과정

`range(1, 11)`은 시작값 `1`은 포함하고 끝값 `11`은 포함하지 않는다. 즉 `i`는 `1`부터 `10`까지 변한다.

|반복 구간|`i`|누적 `hap`|
|---|---:|---:|
|처음|1|1|
|중간|5|15|
|마지막|10|55|

반복이 끝났을 때 마지막으로 대입된 `i`는 `10`이고, `hap`은 `1 + ... + 10 = 55`다. Python의 `for`문은 별도의 블록 스코프를 만들지 않으므로, 반복문 밖에서도 `i`를 사용할 수 있다. 단, 리스트 컴프리헨션의 반복 변수는 바깥으로 새지 않는다는 점은 구분한다.

> 원문 메모: **파이썬에서는 for문 안에서 만든 변수도 for문 밖에서 사용할 수 있다.**

## 문제 20 — 리스트로 완전 이진트리 만들기와 재귀

```python
class Node:
    def __init__(self, value):
        self.value = value
        self.children = []

def tree(li):
    nodes = [Node(i) for i in li]
    for i in range(1, len(li)):
        nodes[(i - 1) // 2].children.append(nodes[i])
    return nodes[0]

def calc(node, level=0):
    if node is None:
        return 0
    return (node.value if level % 2 == 1 else 0) + sum(
        calc(n, level + 1) for n in node.children
    )

li = [3, 5, 8, 12, 15, 18, 21]
root = tree(li)
print(calc(root))
```

### 정답

```text
13
```

### 풀이 과정

`nodes[(i - 1) // 2]`는 배열로 완전 이진트리를 만들 때 부모의 인덱스를 구하는 식이다. `i = 1, 2`의 부모는 `0`, `i = 3, 4`의 부모는 `1`, `i = 5, 6`의 부모는 `2`가 된다.

원문에 있던 트리 메모를 구조에 맞게 옮기면 다음과 같다.

```text
        3          level 0: 짝수 → 더하지 않음
      /   \
     5     8        level 1: 홀수 → 5 + 8을 더함
    / \   / \
   12 15 18 21     level 2: 짝수 → 더하지 않음
```

`calc`은 현재 `level`이 홀수일 때만 `node.value`를 더하고, 자식에게는 `level + 1`을 전달한다. 따라서 루트 `3`과 리프 `12, 15, 18, 21`은 제외되고, 레벨 1의 `5 + 8`만 남아 `13`이 출력된다.

## 문제 23 — 기본 매개변수

```python
def func(num1, num2=2):
    print('a =', num1, 'b =', num2)

func(20)
```

### 정답

```text
a = 20 b = 2
```

### 풀이 과정

호출식 `func(20)`은 첫 번째 인수만 전달한다. `num1`에는 `20`이 들어가고, 전달받지 못한 `num2`에는 선언 시 정한 기본값 `2`가 사용된다. `print`는 인수 사이에 기본 구분자 공백 하나를 넣으므로 정확한 출력은 `a = 20 b = 2`다.

## 문제 43 — `return` 들여쓰기와 슬라이싱

```python
def cnt(str, p):
    result = 0
    for i in range(len(str)):
        sub = str[i:i+len(p)]
        if sub == p:
            result += 1
        return result

str = "abdcabcabca"
p1 = "ca"
p2 = "ab"
print(f'ab{cnt(str, p1)} ca{cnt(str, p2)}')
```

### 정답

```text
ab0 ca1
```

### 풀이 과정

핵심은 `return result`가 `for`문 **안쪽에 들여쓰기되어 있다는 점**이다. 따라서 반복을 한 번만 하고 곧바로 반환한다.

|호출|첫 반복의 `i`|`sub = str[0:2]`|비교|반환값|
|---|---:|---|---|---:|
|`cnt(str, p1)`|0|`"ab"`|`"ab" == "ca"`는 거짓|0|
|`cnt(str, p2)`|0|`"ab"`|`"ab" == "ab"`는 참 → `result += 1`|1|

마지막 f-string의 글자 `ab`와 `ca`는 단지 고정 문자열이다. 첫 호출은 `p1 = "ca"`의 결과이고 둘째 호출은 `p2 = "ab"`의 결과라서, 출력이 직관적인 라벨 순서와 달라 보여도 `ab0 ca1`이 맞다. 만약 `return result`를 `for`문 밖으로 한 칸 내리면 문자열 전체에서 겹치는 경우까지 세는 완전히 다른 함수가 된다.

## 시험 대비 Python 문법·주의사항

### 1. 슬라이싱: `lst[::-2]` 읽기

슬라이싱은 `시작:끝:간격`이고, 끝 인덱스는 포함하지 않는다.

```python
lst = [0, 1, 2, 3, 4, 5]
lst[1:5:2]   # [1, 3] : 1부터 5 직전까지 2칸씩
lst[::-1]    # [5, 4, 3, 2, 1, 0] : 역순
lst[::-2]    # [5, 3, 1] : 끝에서 시작해 2칸씩 역순
```

음수 간격일 때 생략한 시작은 맨 끝, 생략한 끝은 맨 앞 바깥으로 해석한다. `s[i:i+len(p)]`처럼 범위를 넘어도 오류가 아니라 가능한 부분 문자열만 반환한다.

### 2. 논리값과 비교

- 논리 리터럴은 대문자로 시작하는 `True`, `False`다. `true`, `false`는 이름 오류다.
- `==`는 값 비교, `is`는 같은 객체인지(정체성) 비교다. 숫자·문자열 값을 비교할 때 `is`를 쓰지 않는다.
- `0`, `0.0`, `''`, `[]`, `{}`, `set()`, `None`, `False`는 조건식에서 거짓이고 그 밖의 값은 참이다.
- `and`, `or`는 단락 평가한다. 결과가 꼭 `True`/`False`가 아니라 마지막으로 평가한 피연산자일 수도 있다.

```python
bool([])        # False
3 and 5         # 5
0 or '대체값'    # '대체값'
```

### 3. `range`, 나눗셈, 대입

- `range(start, stop, step)`에서 `stop`은 제외한다. `range(5)`는 `0`~`4`다.
- `/`는 실수 나눗셈, `//`는 바닥 나눗셈, `%`는 나머지다. 음수의 `//` 결과도 바닥 방향이라는 점을 주의한다.
- `a = b = 2`는 두 이름에 같은 값 `2`를 바인딩한다. 가변 객체에서는 `a = b = []`처럼 쓰면 같은 리스트를 공유하므로 주의한다.

### 4. 함수·리스트에서 자주 나오는 함정

- 기본 매개변수는 인수를 생략했을 때만 적용된다.
- 기본값으로 `[]`, `{}` 같은 가변 객체를 두면 호출 사이에 값이 누적될 수 있다. 보통 `None`을 기본값으로 둔다.
- `list.append(x)`는 리스트를 제자리에서 바꾸고 반환값은 `None`이다.
- 함수 안에서 대입한 이름은 기본적으로 지역 변수다. 바깥 이름을 바꾸려면 `global`/`nonlocal`을 명시해야 한다.
- 들여쓰기가 곧 블록 경계다. 특히 `return`, `break`, `continue`의 들여쓰기 한 칸이 출력 전체를 바꾼다.
