# 정보처리기사 실기 코드 문제 — 공통 풀이법과 연산자 메모

이 문서는 C·Java·Python 문제를 풀 때 공통으로 쓰는 추적 방법을 모은 노트다. 언어별 문법과 선택 문제의 상세 풀이는 각각 `정보처리기사_실기_정리_C.md`, `정보처리기사_실기_정리_Java.md`, `정보처리기사_실기_정리_Python.md`에서 확인한다.

## 1. 코드 출력 문제를 푸는 순서

1. **초기값을 표로 적는다.** 선언부와 대입문을 만났을 때만 값을 갱신한다.
2. **식을 괄호로 복원한다.** 우선순위가 섞인 식은 한 줄로 암산하지 않는다.
3. **조건식은 참/거짓과 실제 값 둘 다 적는다.** C·Java에서는 참이 `1` 또는 `true`라는 결과와, `&&`/`||`의 단락 평가 여부를 구별한다.
4. **반복문은 첫 반복·마지막 반복을 확인한다.** 특히 `range`의 끝값, 배열의 마지막 인덱스, `break` 뒤의 흐름을 확인한다.
5. **함수는 호출 시점의 인수와 반환값을 분리해 적는다.** 매개변수 변경이 호출자 변수에도 반영되는지는 언어와 자료형에 따라 다르다.
6. **마지막 출력문만 다시 읽는다.** 값이 맞아도 공백, 줄바꿈, 문자열 리터럴이 다르면 답이 달라진다.

## 2. 연산자 우선순위 암기: `괄 증 단 산 시 관 논 대`

이 암기법은 C와 Java의 계산식에 특히 유용하다. 아래 순서에서 **위일수록 먼저** 계산한다. 같은 우선순위는 보통 왼쪽에서 오른쪽으로 계산하지만, 대입과 삼항 연산자는 예외적으로 오른쪽 결합이다.

|기억어|주요 연산자|풀이 때 할 일|
|---|---|---|
|괄|`()`|가장 안쪽 괄호부터 계산한다.|
|증|`++`, `--`|전위는 먼저 증감 후 사용, 후위는 먼저 사용 후 증감이다.|
|단|`!`, `~`, 단항 `+/-`, 형변환|피연산자 하나에 붙는 연산이다.|
|산|`* / %`, 그 다음 `+ -`|정수 나눗셈/나머지를 특히 확인한다.|
|시|`<<`, `>>` (`>>>`는 Java)|비트 이동은 산술 연산 다음이다.|
|관|`< <= > >=`, `== !=`|비교 결과를 논리값으로 바꾼다.|
|논|`&` → `^` → `\|` → `&&` → `\|\|`|비트/논리 연산의 세부 순서를 다시 나눈다.|
|대|`?:`, `=`, `+=` 등|삼항 연산자와 대입은 마지막에 처리한다.|

`관`에는 실제로 관계 연산자와 동등 비교 연산자가 모두 들어간다는 뜻으로 사용했다. 예를 들어 C의 아래 식은 다음처럼 복원한다.

```c
a * 3 + b > d || c - b / a <= d && 1
((a * 3 + b) > d) || (((c - (b / a)) <= d) && 1)
```

### `&`, `^`, `|`와 `&&`, `||` 구분

|연산자|뜻|단락 평가|대표 피연산자|
|---|---|---|---|
|`&`|비트 AND / 논리 AND|없음: 양쪽 모두 평가|정수, C·Java의 boolean|
|`^`|비트 XOR / 논리 XOR|없음|정수, C·Java의 boolean|
|`&#124;`|비트 OR / 논리 OR|없음: 양쪽 모두 평가|정수, C·Java의 boolean|
|`&&`|논리 AND|왼쪽이 거짓이면 오른쪽 미평가|조건식|
|`&#124;&#124;`|논리 OR|왼쪽이 참이면 오른쪽 미평가|조건식|

`i++ | j++`처럼 증감식이 섞이면 `|`는 오른쪽도 실행한다. 반면 `i++ || j++`는 왼쪽 값이 참이면 `j++`를 실행하지 않는다. Python은 `&&`, `||` 대신 `and`, `or`를 쓴다.

## 3. 자주 놓치는 공통 체크 항목

- **배열 인덱스는 0부터** 시작한다. 길이가 `n`이면 마지막 인덱스는 `n - 1`이다.
- **문자열 끝**: C 문자열은 `\0` 앞까지가 내용이다. 다른 언어의 `length`/`len`은 끝 문자까지 세지 않는다.
- **정수 나눗셈**: C와 Java에서 정수끼리 `/`하면 소수점 아래가 버려진다. Python의 `/`는 실수 나눗셈, `//`는 몫(바닥 나눗셈)이다.
- **출력 형식**: `print(a, b)`는 기본적으로 사이에 공백을 넣고, C의 `printf`와 Java의 `print`는 작성한 문자열만 출력한다.
- **원문 코드의 오타**: 시험 문제에서는 오타가 실제 문법 오류인지, 단순 표기 오류인지 구분해야 한다. 예를 들어 C의 `deafult:`는 `default:`가 아니라 단순 레이블이라 컴파일될 수 있으며, 실행 흐름도 달라진다.
