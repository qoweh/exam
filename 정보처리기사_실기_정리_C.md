# 정보처리기사 실기 C — 선택 문제와 시험 문법 노트

원문 번호 `2, 3, 5, 7, 11, 14, 20, 30, 40, 48, 54`를 정리했다. 54번은 7번과 코드가 완전히 같아서 7번에 통합했다. 각 출력은 C의 일반적인 정수 표현과 원문 코드의 실행 흐름을 기준으로 한다.

## 문제 2 — 우선순위가 섞인 논리식

```c
#include <stdio.h>

main()
{
    int a = 5, b = 10, c = 15, d = 30, result;
    result = a * 3 + b > d || c - b / a <= d && 1;
    printf("%d\n", result);
}
```

### 정답

```text
1
```

### 풀이 과정

원문 메모의 `괄 증 단 산 시 관 논 대` 순서를 적용해 식을 먼저 괄호로 복원한다.

```c
((a * 3 + b) > d) || (((c - (b / a)) <= d) && 1)
```

|부분식|계산|결과|
|---|---|---:|
|`a * 3 + b`|`5 * 3 + 10`|25|
|`25 > d`|`25 > 30`|거짓(0)|
|`b / a`|정수 나눗셈 `10 / 5`|2|
|`c - 2 <= d`|`15 - 2 <= 30`|참(1)|
|`1 && 1`|둘 다 참|1|
|`0 || 1`|한쪽이 참|1|

따라서 `result`는 1이고 `%d`로 `1`을 출력한다. `&&`가 `||`보다 우선이므로 오른쪽의 `... <= d && 1`이 먼저 하나의 덩어리로 계산되는 것이 핵심이다.

### 원문 메모·사진

```text
((a * 3 + b) > d) || (((c - (b / a)) <= d) && 1)
괄 증 단 산 시 관 논 대
```

## 문제 3 — 복합 대입 연산자

```c
#include <stdio.h>

main()
{
    int i = 10, j = 10, k = 30;
    i /= j;
    j -= i;
    k %= j;
    printf("%d, %d, %d\n", i, j, k);
}
```

### 정답

```text
1, 9, 3
```

### 풀이 과정

복합 대입 `op=`는 왼쪽 변수를 한 번 갱신한다. 앞 줄에서 바뀐 값은 다음 줄에 그대로 사용한다.

|실행문|같은 뜻|실행 뒤 `(i, j, k)`|
|---|---|---|
|초기화|—|`(10, 10, 30)`|
|`i /= j;`|`i = i / j;` → `10 / 10`|`(1, 10, 30)`|
|`j -= i;`|`j = j - i;` → `10 - 1`|`(1, 9, 30)`|
|`k %= j;`|`k = k % j;` → `30 % 9`|`(1, 9, 3)`|

마지막 `printf`의 쉼표와 공백도 문자열에 있으므로 `1, 9, 3`이 출력된다.

## 문제 5 — `scanf`의 8진수·16진수 입력

```c
#include <stdio.h>

main() {
    int i, j;
    scanf("%o#%x", &i, &j);
    printf("%d %d", i, j);
}
```

### 정답

입력 형식은 `<8진수>#<16진수>`이며, 출력은 두 값을 10진수로 보여 준다. 예를 들어 입력이 `10#10`이면 출력은 다음과 같다.

```text
8 16
```

### 풀이 과정

서식 문자열의 `#`은 특별한 변환 문자가 아니라 입력에 실제로 있어야 하는 문자다.

|입력 부분|`scanf` 서식|해석|저장값|
|---|---|---|---:|
|`10`|`%o`|8진수 `10`|8|
|`#`|`#`|문자 `#`와 일치해야 함|—|
|`10`|`%x`|16진수 `10`|16|

그 뒤 `%d`는 저장된 정수 값을 10진수로 출력한다. `scanf`에서 변수 앞에 `&`를 붙인 이유는 함수가 값을 저장할 **주소**를 받아야 하기 때문이다.

## 문제 7 — 삼항·비트·논리 연산자

```c
#include <stdio.h>

int main() {
    int x = 7, y = 4, z;
    z = y % 3 < 3 ? 2 : 1;
    z = z & z >> 1;
    z = x > 5 && z <= 3 ? z * x : z / x;
    printf("%d", z);
    return 0;
}
```

### 정답

```text
0
```

### 풀이 과정

|순서|식의 복원|계산|`z`|
|---|---|---|---:|
|1|`(y % 3) < 3 ? 2 : 1`|`4 % 3 = 1`, `1 < 3`은 참|2|
|2|`z & (z >> 1)`|`2`는 이진수 `10`, `2 >> 1 = 1`; `10 & 01 = 00`|0|
|3|`(x > 5 && z <= 3) ? z * x : z / x`|`7 > 5`, `0 <= 3` 모두 참 → `0 * 7`|0|

두 번째 줄에서 `>>`가 `&`보다 우선이므로 `z & (z >> 1)`이다. 세 번째 줄은 `&&` 조건 전체가 삼항 연산자의 조건이 된다. 최종값 0이 출력된다.

> 중복 처리: 원문 **54번**은 위 코드와 완전히 같으므로 별도 문제로 반복 수록하지 않았다.

## 문제 11 — 숫자 뒤집기

```c
#include <stdio.h>

int main() {
    int number = 1234;
    int div = 10, result = 0;
    while (number != 0) {
        result = result * div;
        result = result + number % div;
        number = number / div;
    }
    printf("%d", result);
}
```

### 정답

```text
4321
```

### 풀이 과정

`number % 10`으로 맨 오른쪽 자리 숫자를 꺼내고, `result * 10`으로 기존 결과를 한 자리 왼쪽으로 민 뒤 그 숫자를 붙인다. `number / 10`은 정수 나눗셈이라 맨 오른쪽 자리를 버린다.

|반복 전 `number`|`result * 10` 뒤|붙이는 `number % 10`|반복 후 `result`|반복 후 `number`|
|---:|---:|---:|---:|---:|
|1234|0|4|4|123|
|123|40|3|43|12|
|12|430|2|432|1|
|1|4320|1|4321|0|

`number`가 0이 되는 순간 반복을 끝내므로 출력값은 `4321`이다.

## 문제 14 — `switch`의 fall-through

```c
#include <stdio.h>

main() {
    int c = 1;
    switch (3) {
        case 1: c += 3;
        case 2: c++;
        case 3: c = 0;
        case 4: c += 3;
        case 5: c -= 10;
        default: c--;
    }
    printf("%d", c);
}
```

### 정답

```text
-8
```

### 풀이 과정

`switch (3)`은 `case 3`으로 점프한다. 그 아래에 `break`가 하나도 없으므로 일치한 case 뒤의 문장을 끝까지 연속 실행한다. 이를 fall-through라고 한다.

|실행 위치|계산|`c`|
|---|---|---:|
|초기값|—|1|
|`case 3`|`c = 0`|0|
|`case 4`|`c += 3`|3|
|`case 5`|`c -= 10`|-7|
|`default`|`c--`|-8|

따라서 `-8`을 출력한다. `default`는 매칭 case가 없을 때만 쓰이는 것처럼 보이지만, 이 문제처럼 위에서 흘러 내려오면 실행될 수 있다.

## 문제 20 — 포인터 배열 역참조

```c
#include <stdio.h>

int main() {
    int* array[3];
    int a = 12, b = 24, c = 36;
    array[0] = &a;
    array[1] = &b;
    array[2] = &c;
    printf("%d", *array[1] + **array + 1);
}
```

### 정답

```text
37
```

### 풀이 과정

`array`는 정수 포인터 3개를 담는 배열이다. 따라서 다음처럼 연결된다.

```text
array[0] → a(12)
array[1] → b(24)
array[2] → c(36)
```

- `*array[1]`은 `*(array[1])`이다. `array[1]`이 `&b`이므로 값은 `24`다.
- `**array`는 `*(*(array + 0))`, 즉 `*(array[0])`이다. 값은 `a`, 곧 `12`다.
- 따라서 `24 + 12 + 1 = 37`이다.

배열 첨자는 역참조보다 먼저 묶이므로 `*array[1]`을 `(*array)[1]`로 읽으면 안 된다.

## 문제 30 — 같은 전역 배열 주소를 반환하는 함수

```c
#include <stdio.h>

char n[30];
char* getname() {
    printf("이름 입력 : ");
    gets(n);
    return n;
}

main() {
    char* n1 = getname();
    char* n2 = getname();
    char* n3 = getname();
    printf("%s\n", n1);
    printf("%s\n", n2);
    printf("%s\n", n3);
}
```

### 정답

세 번 입력한 값이 각각 `첫째`, `둘째`, `셋째`라면, 화면에는 입력 안내 뒤 다음이 출력된다.

```text
셋째
셋째
셋째
```

### 풀이 과정

`n`은 `getname` 함수 안에서 새로 만들어지는 배열이 아니라 프로그램 전체에서 하나뿐인 전역 배열이다. `return n;`은 문자열 복사본이 아니라 그 배열의 첫 주소를 반환한다.

|호출|전역 배열 `n`에 남은 내용|`n1`, `n2`, `n3`가 가리키는 곳|
|---|---|---|
|첫 번째|`첫째`|`n1` → 같은 `n`|
|두 번째|`둘째`로 덮어씀|`n1`, `n2` → 같은 `n`|
|세 번째|`셋째`로 덮어씀|`n1`, `n2`, `n3` → 같은 `n`|

그래서 세 포인터 모두 마지막 입력값을 출력한다. `gets`는 버퍼 길이를 검사하지 않아 위험하며 C11에서 제거된 함수다. 실제 프로그램은 `fgets(n, sizeof n, stdin)`을 써야 하지만, 이 문제의 출력 추론에서는 원문 그대로 같은 배열을 공유한다는 점이 핵심이다.

## 문제 40 — 구조체, 함수, 비트 AND

```c
#include <stdio.h>

typedef struct student {
    char* name;
    int score[3];
} Student;

int dec(int enc) {
    return enc & 0xA5;
}

int sum(Student* p) {
    return dec(p->score[0]) + dec(p->score[1]) + dec(p->score[2]);
}

int main() {
    Student s[2] = { "Kim", {0xA0, 0xA5, 0xDB}, "Lee", {0xA0, 0xED, 0x81} };
    Student* p = s;
    int result = 0;
    for (int i = 0; i < 2; i++) {
        result += sum(&s[i]);
    }
    printf("%d", result);
    return 0;
}
```

### 정답

```text
908
```

### 풀이 과정

`dec`는 값과 `0xA5`의 비트 AND만 남긴다. `p->score[k]`는 `(*p).score[k]`와 같은 뜻이다. 각 학생의 세 점수를 먼저 계산한다.

|학생|원본 값|`enc & 0xA5` 결과|부분합|
|---|---|---|---:|
|Kim|`0xA0`, `0xA5`, `0xDB`|`0xA0 = 160`, `0xA5 = 165`, `0x81 = 129`|454|
|Lee|`0xA0`, `0xED`, `0x81`|`0xA0 = 160`, `0xA5 = 165`, `0x81 = 129`|454|

반복문은 `sum(&s[0])`, `sum(&s[1])`을 차례대로 더한다. 따라서 `result = 454 + 454 = 908`이다.

> 개인 메모 반영 — 이 문제는 계산기로 한 번 검산하기 좋다. 예: `0xDB & 0xA5 = 0x81`, `0xED & 0xA5 = 0xA5`. 다만 시험에서는 16진수를 2진수 네 자리씩(`D=1101`, `B=1011` 등)으로 펼쳐 AND하는 방법도 함께 익힌다.

## 문제 48 — 값 호출과 `deafult:` 레이블

```c
#include <stdio.h>

void swap(int a, int b) {
    int t = a;
    a = b;
    b = t;
}

int main() {
    int a = 11;
    int b = 19;
    swap(a, b);
    switch(a) {
        case 1:
            b += 1;
        case 11:
            b += 2;
        deafult:
            b += 3;
            break;
    }
    printf("%d", a-b);
}
```

### 정답

```text
-13
```

### 풀이 과정

`swap(a, b)`의 매개변수 `a`, `b`는 호출자의 변수와 별개인 지역 변수다. 함수 안에서만 교환되고 `main`의 `a = 11`, `b = 19`는 그대로다.

그다음 `switch(a)`는 `case 11`으로 점프한다. 원문은 `default:`가 아니라 **`deafult:`**라고 썼다. C에서는 이것이 문법 오류가 아니라 이름이 `deafult`인 일반 레이블이다. 따라서 `case 11` 뒤에서 아래로 흘러 실행된다.

|순서|실행|`a`|`b`|
|---|---|---:|---:|
|함수 호출 뒤|값 호출이므로 호출자 값 유지|11|19|
|`case 11`|`b += 2`|11|21|
|`deafult:` 아래|`b += 3`|11|24|
|출력식|`a - b = 11 - 24`|—|-13|

`break`는 `b += 3` 뒤에 있으므로 여기서 switch를 끝낸다. 철자가 올바른 `default:`였더라도 `case 11`에서 break가 없으므로 아래 문장이 실행되어 이 입력에서는 같은 수치가 나오지만, 매칭 case가 없을 때의 동작은 달라진다.

## 시험 대비 C 문법·주의사항

### 1. 포인터 배열과 배열 포인터

별표와 대괄호가 섞이면 **변수 이름부터 안쪽→바깥쪽**으로 읽는다.

|선언|읽는 법|핵심|
|---|---|---|
|`int *arr[3];`|`arr`은 길이 3의 배열, 각 원소는 `int *`|포인터 **배열**|
|`int (*p)[3];`|`p`는 `int[3]` 배열 하나를 가리키는 포인터|**배열 포인터**|
|`int **(*p)[3];`|`p`는 `int **` 원소 3개짜리 배열을 가리키는 포인터|괄호가 `p`를 먼저 묶음|
|`int **(***p)[3];`|`p`는 세 단계 포인터를 거쳐 `int **` 원소 3개짜리 배열에 이르는 선언|복잡한 선언도 `p`에서 밖으로 읽음|

`int *arr[3]`의 `arr + 1`은 포인터 한 칸(주소 크기) 이동하지만, `int (*p)[3]`의 `p + 1`은 `int` 3개짜리 배열 한 덩어리만큼 이동한다. `*array[1]`은 `*(array[1])`이고, `(*array)[1]`과 다르다는 점을 문제 20에서 확인했다.

### 2. 문자열·입출력

- C 문자열 끝에는 보이지 않는 널 문자 `\0`이 있다. `while (str[i] != '\0')`은 길이를 세는 전형적인 코드다.
- `scanf("%d", &x)`처럼 숫자 변수에는 주소 연산자 `&`를 붙인다. 배열 이름 `char s[20]`은 대부분의 식에서 첫 원소 주소로 변환되므로 `scanf("%s", s)`처럼 쓴다.
- `%d`(10진 정수), `%o`(8진), `%x`(16진), `%c`(문자), `%s`(문자열)을 구분한다.
- `gets`는 사용하지 않는다. 버퍼 길이를 제한하는 `fgets`가 안전하다.

### 3. 제어문·함수의 함정

- `switch`는 일치한 `case`부터 `break`까지 계속 실행한다. `default`도 fall-through 경로에서 실행될 수 있다.
- C는 기본적으로 값 호출이다. `swap(a, b)`가 호출자 값을 바꾸려면 `void swap(int *a, int *b)`처럼 주소를 받고 `*a`, `*b`를 바꿔야 한다.
- `&&`, `||`는 단락 평가하지만 `&`, `|`는 양쪽을 모두 평가한다.
- 현대 C에서는 `int main(void)` 또는 `int main(int argc, char *argv[])`를 쓴다. 원문처럼 반환형을 생략한 `main()`은 오래된 교재 표기이며 현대 컴파일러에서는 경고/오류가 날 수 있다.
